# daily-summary

Claude Code 대화를 분석하여 일일 작업 요약 리포트를 생성한다.

- 인자 없이 실행: 오늘 날짜 대상
- 날짜 인자 전달: 해당 날짜 대상 (예: `/daily-summary 2026-03-18`)

## 실행 절차

### 1. 날짜 결정

`$ARGUMENTS`가 비어 있으면 오늘 날짜를, 값이 있으면 해당 날짜를 사용한다.

- 형식: `YYYY-MM-DD`

### 2. 세션 데이터 수집

아래 스크립트를 실행하여 해당 날짜의 세션 데이터를 JSON으로 추출한다.

```bash
# 오늘 날짜
python3 ~/.claude/skills/daily-summary/scripts/collect_sessions.py

# 특정 날짜
python3 ~/.claude/skills/daily-summary/scripts/collect_sessions.py 2026-03-18
```

> 출력 스키마:
> ```json
> [
>   {
>     "project": "프로젝트명",
>     "start": "HH:MM",
>     "message_count": 12,
>     "messages": [
>       { "role": "user|assistant", "ts": "HH:MM", "text": "최대 300자" }
>     ]
>   }
> ]
> ```

### 3. 주제 그룹화

수집된 세션 전체를 읽고, **작업 주제 단위**로 묶는다.

- 주제란 "무엇을 만들거나 해결하려 했는가"를 기준으로 한다
  - 좋은 예: `Jira-Notion 자동화 설계`, `Git rebase 오류 디버깅`, `API 응답 스키마 설계`
  - 나쁜 예: `오전 작업`, `세션 2`, `프로젝트 A`
- 하나의 세션 안에 주제가 여러 개일 수 있고, 반대로 여러 세션이 같은 주제일 수 있다
- 주제 수는 3~7개가 적당하다. 지나치게 잘게 쪼개거나 하나로 뭉치지 않는다

### 4. 리포트 작성

아래 형식을 그대로 따른다. **톤은 업무 일지와 개발 블로그의 중간**으로 작성한다.
건조한 나열이 아니라 흐름이 느껴지게, 그러나 감상적이지 않게.

## 📅 {YYYY-MM-DD} 작업 일지

> {오늘 하루를 한 문장으로 — 무엇을 주로 했고 어디까지 닿았는지}
> 예: "자동화 파이프라인 설계를 마무리하고, 예상치 못한 API 스펙 이슈로 오후를 전부 썼다."

**총 대화** {N}건 · **작업 시간대** {HH:MM} ~ {HH:MM} · **프로젝트** {A, B, C}

---

### 🗂 작업 주제별 정리

각 주제를 아래 구조로 작성한다:

#### {이모지} {주제명}

`{시작 시각} ~ {종료 시각}` · `{관련 프로젝트명}`

{2~4문장으로 서술. 무엇을 시도했고, 어떤 문제가 있었고, 어떻게 해결했는지 또는 어디서 막혔는지를 흐름으로 기술한다. 결과가 있으면 결과까지.}

- **핵심 진전**: {이 주제에서 실제로 이뤄진 것 — 완성, 설계 확정, 원인 파악 등}
- **막힌 지점**: {해결 못 한 부분, 우회한 부분} ← 없으면 생략

---

{주제 수만큼 반복}

---

### 💡 오늘의 핵심 인사이트

오늘 대화에서 나온 중요한 판단, 설계 결정, 새로 알게 된 사실을 목록으로 정리한다.
단순 요약이 아닌, "왜 그 결정을 내렸는가"까지 한 줄로 담는다.

- {인사이트 1} — {근거 또는 맥락}
- {인사이트 2} — {근거 또는 맥락}
- ...

---

### 🔖 미완료 · 다음에 이어할 것들

오늘 시작했지만 끝내지 못한 것, 또는 대화 중 "나중에 하자"고 미룬 것들을 정리한다.
우선순위를 느낌 기반으로 High / Mid / Low로 표시한다.

| 우선순위 | 항목 | 맥락 |
|--------|------|------|
| 🔴 High | {항목} | {왜 급한지} |
| 🟡 Mid  | {항목} | {관련 작업} |
| 🟢 Low  | {항목} | {언제 해도 되는 이유} |

항목이 없으면 "오늘 시작한 작업은 모두 마무리됨" 으로 표기한다.

---

### 📊 시간별 대화량

메시지가 있는 시간대만 표시. `█` 1개 = 1건 (user 메시지 기준).

```
09시 ████ 4
10시 ██████████ 10
11시 ███ 3
```

### 주의사항

- `assistant` 응답 중 `tool_use`, `tool_result` 타입은 텍스트 추출 대상에서 제외 (스크립트가 처리)
- `system-reminder` 등 시스템 메시지는 제외 (스크립트가 처리)
- 사용자 메시지가 비어 있거나 `<local-command-caveat>` 포함 시 제외 (스크립트가 처리)
- 현재 프로젝트에 한정하지 않고 해당 날짜에 수정된 **모든 프로젝트**를 포함
- 세션이 하나도 없으면 "해당 날짜에 기록된 대화가 없습니다."로 안내
- 주제 그룹화는 세션 파일 경계를 무시하고 **내용** 기준으로 한다
